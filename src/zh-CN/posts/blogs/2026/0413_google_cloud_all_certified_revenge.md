## 前言

至今，我在is开发者网站撰写了有关AWS认证的文章，并于2026年3月26日发表了题为[《Google Cloud认证全制霸！……在只差一步就被弹回的真实轨迹》](/blogs/2026/03/26/google_cloud_all_certified/)的文章。

在上一篇文章中，我讲述了在大约两个月的疯狂考试冲刺中，虽然力求全冠，但在最后一门“Professional Security Operations Engineer（PSOE）”考试中惜败，无败纪录划上了句号。

然而这次，我于2026年4月11日挑战了PSOE的复仇考试，并顺利通过！至此，我终于达成了目标 **“Google Cloud认证全冠”**。

本文将总结突破PSOE这一难关的经历、上一篇文章未曾提及的“再考试政策（Retake Policy）”、PearsonVUE迁移所带来的变化，以及今后的目标。

## 期待已久的「PSOE 日语版」发布与学习方法重新检视

上次1月31日参加考试时，PSOE仅提供英文版本。由于试卷涉及安全领域特有的特殊工具，并且需要以英语解析复杂的故障排除场景，这成为我失败的主要原因。

因此，我利用官方学习平台「Google Cloud Skills Boost」，通过实践操作，将策略调整为“首先以日语理解概念和应对方法”来进行备考。

就在此时，传来了喜讯。PSOE的日语版竟在**2026年3月31日发布**。我在上一篇文章中曾吐槽，迁移到PearsonVUE时似乎不会有日语化的迹象，但这次正好赶上黄金时机推出。除了在Skills Boost上进行实战型动手学习，能以日语把握题目细微语义，也大大助力了我的此次通过。

顺便一提，我在2月以英文艰难通过的「Professional Cloud Database Engineer（PCDBE）」也在**4月7日发布了日语版**。这意味着对于想挑战Google Cloud认证的人来说，**目前所有有效认证均可用日语考试**，是真正极佳的环境！

## 关于Google Cloud认证的“再考试政策”

虽然上一篇文章未提及，但Google Cloud认证设有严格的**再考试政策（Retake Policy）**。在短时间内力求通过时，这一等待期可能成为一大障碍，需要特别注意。

- 第1次不通过：需等待 14 天后才可再考  
- 第2次不通过：需等待 60 天后才可再考  
- 第3次不通过：需等待 365 天后才可再考  

就我而言，由于1月31日的考试未通过，理论上最早可在2月14日之后重考。然而，考虑到演练所用的Udemy教材被部分删除而需调整学习策略，以及对PearsonVUE迁移时可能推出日语版的微弱期待，最终我还是安排了充足的准备时间，于4月11日再次应考。尤其是若连续两次未通过，就得被“60天（约2个月）”的等待期所困，因此在再考前做好充分准备非常重要。

## 瞄准的是“Google Cloud Partner All Certification Holders”

这次，我之所以坚持完成Google Cloud认证全冠，其中一个原因是**“[Google Cloud Partner All Certification Holders](https://cloud.google.com/blog/ja/topics/partners/google-cloud-partner-all-certification-holders/)”**这一表彰项目的存在。该项目会公布所有持有Google Cloud认证资格的人，并由Google Cloud Japan向合作伙伴企业正式表达感谢之意。最近一次获奖者名单是在2025年11月公布的。

我暗自将一个小目标定为：若将来有机会让公司作为Google Cloud合作伙伴注册，到那时能以“All Certification Holders”的身份正式入榜。

## 随PearsonVUE迁移带来的变化与注意事项

从这次考试开始，试卷分发提供商由Kryterion迁移至PearsonVUE。实际考试后，我发现了一些重大变化和心得，现一并分享。

**1. 可选考试中心增多，考试时间变更灵活性提升**  
在我所在地区，Kryterion时代只能选择特定的一个考试中心，而更换为PearsonVUE后，考生可选择多个考试中心。另外，在Kryterion下考试时间的变更需在72小时前锁定（虽可支付手续费更改），但在PearsonVUE中可在考试前24小时内进行更改，这对考生而言实乃一大利好。

**2. 考试系统更易用（划除功能）**  
在PearsonVUE的考试系统中，对选项文本“右键点击”即可**添加删除线（Strike-through）**。这样可以在视觉上排除明显错误的选项，利用排除法作答变得非常方便。虽然低调，却是相当强大且实用的功能。

**3. 合格后各类通知速度显著加快**  
令我个人非常高兴的是，合格后的正式通知发送速度大幅提升。此前，Credly徽章发放通知约在“次日6点左右”，Google Cloud的合格确认通知约在“次翌日5点左右”，而现在则大幅提前。  
- Credly徽章发放通知：次日6点左右 → **考试结束约1小时后**  
- Google Cloud合格确认通知：次翌日5点左右 → **次日4点左右**  
能立即感受到正式合格，让考后不安的等待时间大大缩短，真是令人欣喜的改进。

**4. 【注意】全冠达成后PearsonVUE入口将消失**  
这是由于全冠达成（所有考试均通过）的特殊情况所埋下的坑：当没有未获取且可报名的考试后，Google Cloud认证门户将不再出现跳转至PearsonVUE仪表盘的入口。  
而问题在于，一旦无法登录PearsonVUE个人页面，就**无法进行“收据再发行（邮件重发）”**的操作。特别是当“报名后更改了考试日期”时，最初收到的报名收据与实际考试日期不符。即便之后想重新获取标注正确考试日期的收据，也无法进入门户，可能会在公司报销时遇到麻烦。

顺带一提，对于包含日本发票制度所需“合格账单发行事业者注册号码”的收据，PearsonVUE方面可以另外提供发行。

顺便提一下，在同样通过PearsonVUE分发的AWS认证门户中，即便是已取得的考试，也能跳转至PearsonVUE仪表盘，因此不会出现此问题。也许仅是我在Google Cloud门户中遗漏了其他访问步骤，但如果当前行为确为既定方案，也希望Google Cloud方面能像AWS一样进行改进。

**5. 【小贴士】认证证书及有效期日期会暂时显示为美国时间（前一天）**  
或许是迁移至PearsonVUE的影响，考试后即时发放的认证证书的获得日期及有效期，有时会在系统上暂时以美国时间（前一天）登记。这次，我在日本时间“4月11日”考试并通过，但考试当日发放的证书上却标注为“4月10日”，相差一天。不过次日再次查看系统时，已自动修正为正确的日本时间日期（4月11日）。因此，如果有的伙伴在考试结束后发现日期错位，也别着急，等到次日即可。

## 结语

继AWS认证之后，我也完成了Google Cloud认证的全冠。

事实上，还有一项我尚未取得的AWS最新认证「Generative AI Developer - Professional (AIP-C01)」将于4月14日正式发布，因此我计划在4月16日参加该考试。由于此前在Beta版考试中遗憾未通过，这对我而言是AWS方面的**“复仇赛”**。若能顺利通过，就将正式达成“AWS&Google Cloud双全冠”，届时也会另撰文章再行报告。

这次深入系统地学习了两大主流云平台，我深切感受到自己在多云环境下的设计与提案能力显著提升。今后将借此经验，积极撰写**“对比AWS与Google Cloud架构与理念差异的文章”**等内容。

另一方面，由于我在短期内一口气获取了如此多的认证，已开始为**“两年后将迎来大规模集中更新潮的恐惧”**而战栗。事实上去年我在AWS认证中也经历过类似的更新潮，AWS拥有“更新高阶认证即可自动更新相关低阶认证”的出色制度。但可惜Google Cloud没有这项制度，因此**包括低阶认证在内的所有认证都要一个一个重新考试进行更新**。虽然官方也提供了“再认证的简易考试（Recertification）”作为救济措施，但光是想象维护全冠的过程，就让人不寒而栗（笑）。

然后，在完成了AWS和Google Cloud之后，突然脑中浮现一个问题……**“那么，Azure怎么办？”**。事实上，我关于Azure的入门资格「AZ-900」在四年前挑战AWS认证期间就已取得，因此似乎有望实现三大云平台制霸之路，不过考虑到前面提及的恐怖更新潮，也让我稍显犹豫。

暂且就此结束我的Google Cloud认证全冠挑战记录。希望能为之后挑战认证的各位提供参考！
